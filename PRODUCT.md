# Product Strategy: VibeReference

## Vision

The reference site for AI-first builders choosing the tools, workflows, and deployment paths that take a product from idea to launch. VibeReference should reduce tool confusion by organizing the market around real builder decisions, not vendor lists.

## Target User

Solo founders and indie hackers building with AI who need to decide:

- whether they want a CLI, IDE, cloud, self-hosted, or app-builder workflow
- which vendor is best for that lane
- which supporting stack choices they need next (hosting, auth, database, payments)

## Positioning

"Choose the workflow before you choose the vendor." VibeReference is a decision-support reference site for AI-first builders. It explains product categories, then maps the best tools and trade-offs inside each category.

## Current Stage

Live at vibereference.com. Content site. SEO-first distribution. The strongest existing cluster is cloud coding agents; the main gap is consistent taxonomy and deeper coverage of local, self-hosted, and app-builder lanes.

## Current Priorities

1. Publish a canonical agent taxonomy hub that explains surface, hosting, ownership, and focus
2. Expand local, open-source, and self-hosted agent coverage
3. Build a dedicated app-builder cluster for Replit Agent, Bolt, Lovable, and v0
4. Continue SEO growth on "best [tool/workflow] for solo founders" queries
5. Cross-promote with VibeWeek and LaunchWeek

## Key Decisions

- **Content-only**: Reference/informational site. No SaaS features.
- **Editorial model**: Cluster content by builder intent and workflow shape, not by vendor alphabet.
- **Canonical page types**: Reuse a tight set of taxonomy explainers, product family pages, variant pages, comparison pages, and workflow/deployment guides.
- **Agent coverage strategy**: Keep strong cloud-agent coverage, then fill the local/self-hosted gap before adding long-tail vendor pages.
- **Stack**: Next.js, static. No Convex or Clerk.
- **Tier 2 social**: SEO-first. Share via DA hub.
- **Ecosystem play**: VibeReference (choose workflow and tools) → VibeWeek (build with them) → LaunchWeek (launch the product).
